你应该有自己的技术博客

发表于 2017-06-15

一直都没有做自己的技术博客。本想自己搭服务器做的。后来无意中看到hexo可以满足自己的需求。所以找了教程，搭了一个自己的技术博客。下面记录一下过程。

一·安装node和git

node和git的安装不多说了。不懂的可以自行google。

二.github的设置

这步只是创建工程(创建的时候注意Repository name。比如我的Github账号是consolelogwwy，那么我应该创建的Repository name是：consolelogwwy.github.io )。

三.配置和使用GitHub

(1)配置SSh key:
首先我们需要检查你电脑上现有的ssh key,打开git bsah，输入命令：ls -al ~/.ssh ，如果不存在就没有关系，如果存在的话，直接删除.ssh文件夹里面所有文件。
(2)输入以下命令，记住475451510@qq.com是我的邮箱，这个邮箱是你当时注册github时所用的邮箱。
ssh-keygen -t rsa -C “475451510@qq.com”
然后会要求你填写passphrase，一路回车即可。
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):（回车就好），
Enter passphrase (empty for no passphrase):（回车，不需要输入密码）
Enter same passphrase again:（回车不需要输入密码）
注意： 因为Hexo的自动部署如果有密码的话会部署失败
(3)输入：ssh-agent -s 。
(4)继续输入：ssh-add ~/.ssh/id_rsa ，输入之后，在我这里是出错了，不知道你的有没有出错。如何也出错了，就输入：
eval ssh-agent -s ，最后输入： ssh-add 即可。
(5)到了这一步，就可以添加SSH key到你的Github账户了。键入以下指令，拷贝Key（先拷贝了，等一下可以直接粘贴）：
clip < ~/.ssh/id_rsa.pub
(6)然后到Github里面，一次进行以下四个步骤：1.settings –> 2.SSH and GPG keys –>3.New SSH key –>4.填写title(随便填)，黏贴key。
(7)测试是否成功，输入： $ ssh -T git@github.com ，如果出现如下图所示，那么恭喜你，ssh key已经配置好了。

四.配置Hexo进行一键部署安装

(1)打开 Hexo目录下的 _config.yml，以下是修改的地方:
    # Deployment
    ## Docs: http://hexo.io/docs/deployment.html
    deploy:
    type: git     
    repository: git@github.com:consolelogwwy/consolelogwwy.github.io.git
    branch: master    
(2)先进入 Hexo目录，在文件夹任意地方右键 Git Bash Here。
(3)输入：npm install hexo-deployer-git --save  进行安装部署所需要的插件。
(4)继续输入： hexo clean && hexo g && hexo deploy 。
(5)在浏览器输入（consolelogwwy是你的用户名）：http://consolelogwwy.github.io/,博客部署成功!
